This reference outlines the available feature flags within the system.

## Capabilities

Capabilities enable specific features or workflows.

### Self-Service

These features can be enabled or disabled directly by Company Administrators.

| Feature                                 | Description                                                                                                                                                                              |
| --------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Allow item approval before payment**  | With this feature enabled, companies will be able to mark non-guest list items as requiring approval. Customers will be unable to pay for/complete their order until it's been approved. |
| **Extended customer country selection** | When enabled, customers will be able to select any country to use as their address, rather than just countries enabled for the company.                                                  |

## Limitations

<Warning>
  Limitations act as restrictions. When a Limitation flag is **Enabled**, the specific functionality is **Blocked** or **Hidden**.
</Warning>

### Self-Service Restrictions

These restrictions can be toggled by Company Administrators to restrict their own portal functionality.

| Limitation                                                                     | Description                                                                                                                                                                             |
| ------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Force profile questions on checkout**                                        | During checkout profile questions as part of data capture are usually only asked if the data has not been provided. Enabling this will force it to be asked every time.                 |
| **Disallow profile changes**                                                   | Customers will only be able to edit profile fields during the checkout process.                                                                                                         |
| **Disallow data capture editing**                                              | Customers will not be able to edit their data capture answers outside of the checkout process.                                                                                          |
| **Disable instant checkout**                                                   | Hide the instant checkout button from box office.                                                                                                                                       |
| **Force customers to login/register before being able to add items to basket** | When enabled, the 'buy now' button on the website will be replaced with a login button for guest customers.                                                                             |
| **Force customers to login/register before being able to browse the shop.**    | When enabled, customers will be redirected to the login page prior if they are unauthenticated when visiting the site. If SSO is enabled, they will be redirected to your SSO provider. |
| **Disable sitemap generation**                                                 | When enabled, this companies sitemap will not be generated or served on the public internet.                                                                                            |
| **Disable schema JSON**                                                        | When enabled, schema JSON (json+ld) will be suppressed within the shop.                                                                                                                 |
| **Disable customer feedback collection**                                       | Disabling customer feedback collection will disable all of the feedback widgets embedded within certain customer experiences (such as the checkout flow).                               |

### Hub Enforced Restrictions

These restrictions are applied by Platform Administrators and cannot be removed by the Company.

| Limitation                                       | Description                                                                                                                                                                           |
| ------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Block company access when invoice(s) overdue** | When enabled, companies will be blocked from the system until overdue invoices are cleared.                                                                                           |
| **Disallow editing historic events**             | Events that have passed the end time will no longer be editable.                                                                                                                      |
| **Disallow refunding historic events**           | Events that have passed the end time will no longer be refundable.                                                                                                                    |
| **Disallow free items**                          | Free items cannot be created on events.                                                                                                                                               |
| **Mandate organiser details**                    | Organiser details will be mandatory for all events.                                                                                                                                   |
| **Disallow box office mode**                     | Company users will not be able to access box office mode.                                                                                                                             |
| **Disable global event stock**                   | Hide global stock settings for event items (tickets, products, etc.). Items can still be copied (re-assigned) from other events, but stock can only be configured at the event-level. |
| **Disable global epos stock**                    | Hide global stock settings for epos products. Items can still be copied (re-assigned) from other shops, but stock can only be configured at the shop-level.                           |
| **Disallow processing fees**                     | This company will not be able to set processing fees for all item types.                                                                                                              |
| **Disallow reseller fees**                       | This company will not be able to set reseller fees for all item types.                                                                                                                |
| **Disallow resell/assign existing items**        | Companies will not be able to create sale items from existing items (except for season tickets & via event scheduling).                                                               |
| **Cannot amend Szamlazz credentials**            | When enabled, company users will not be allowed to amend their szamlazz credentials once they have been set.                                                                          |
| **Disallow absorbing item fees**                 | When enabled, the company will no longer be able to absorb processing or reseller fees into their prices.                                                                             |
| **Disable usage metrics**                        | When enabled, we will not record platform usage data (such as the frequency at which features are used, and the journey taken by a user to complete a task).                          |
| **Disallow the ability to view shop from admin** | When enabled, the company users will no longer be given options to view the shop from admin.                                                                                          |
| **Hide global system roles**                     | When enabled, company users will not be able to see global system roles (e.g., Box Office User, Scanning User) in admin.                                                              |
| **Disallow refunding reseller fees**             | When enabled, the company will be unable to refund the reseller fee portion of an item. This can prevent loss of reseller fees where the reseller acts as the merchant of record.     |
| **Disable ticket shop**                          | When enabled, the company's ticket shop will be disabled. Child companies with global login disabled cannot be disabled.                                                              |
| **Disable Google Places**                        | When enabled, customer address lookups will be disabled in the shop and customers will be forced to manually type in their address where required.                                    |
