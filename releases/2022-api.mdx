<Update label="12th December 2022" description="">

- Following the introduction of tax breakdowns in shop orders, the tax breakdown details are now provided via [GET /order-items](/api-reference/orders/retrieve-order-items) and [GET /orders](/api-reference/orders/retrieve-orders).
- The currency ID of any collected fees is now included in [GET /order-items](/api-reference/orders/retrieve-order-items) for use when any transactions take place in custom currencies.
- It is now possible to retrieve topup and cashless product purchases via [GET /order-items](/api-reference/orders/retrieve-order-items).
- When retrieving cashless product purchases via [GET /order-items](/api-reference/orders/retrieve-order-items) it's possible to specify that the shop should be included in the response.

</Update>

<Update label="6th December 2022" description="">

- Removed the `sku` parameter from the [POST /basket-items](/api-reference/baskets/add-to-basket) endpoint, in favour of using topup IDs directly (as returned by [GET /sale-items](/api-reference/sale-items/retrieve-sale-items))
- Added a `currencyId` parameter to the [POST /basket-items](/api-reference/baskets/add-to-basket) endpoint to specify the basket currency, rather than relying on the customer's previously selected currency or the company default.
- Added a global `X-NU-CURRENCY-ID` header that will change the selected currency *for that request only*. This is useful for retrieving sale items with the correct pricing.
- Added a new `feeCurrencyId` value to any order responses via [GET /orders](/api-reference/orders/retrieve-orders), which represents the currency any fees should be displayed in. This is significant with cashless products as the fees will be in a companies default currency when cashless is setup to use a token currency.

</Update>

<Update label="5th December 2022" description="">

- Added the `bundledWith` array on [GET /sale-items](/api-reference/sale-items/retrieve-sale-items) to return the list of sale item IDs that come bundled with an item.
- Added the `bundleLeadOrderItemId` to the [GET /orders](/api-reference/orders/retrieve-orders) and `GET /orders/:id` endpoints, to highlight where an order item is being purchased as part of a bundle.

</Update>

<Update label="28th November 2022" description="">

- Added `isCustom` to the currencies returned from [GET /currencies](/api-reference/currencies/retrieve-currencies), to identify which currencies are custom token currencies.
- Added `topupTokenAmount` to the sale item details returned from [GET /sale-items](/api-reference/sale-items/retrieve-sale-items), and `GET /items` to detail how many tokens a topup will add to a users basket. This value will be null if cashless settings do not use a custom token currenciy.

</Update>

<Update label="4th November 2022" description="">

- Added `subunit` to [GET /currencies](/api-reference/currencies/retrieve-currencies) to complement our new support for non-2DP currencies.

</Update>

<Update label="28th October 2022" description="">

- Added UUID to the `initiate wallet download` endpoint to ensure idempotency between requests.

</Update>

<Update label="19th October 2022" description="">

- Allowed the transaction source to be recorded for cashouts via the `POST /wallets/cashouts` endpoint.
- Allowed the transaction source to be recorded for topups and cashouts via the [POST /orders/upload](/api-reference/cashless/upload-orders) endpoint.

</Update>

<Update label="19th September 2022" description="">

- Added a global response header to all requests: `X-Runtime` which returns the total execution time, in seconds, for any given API in call. This value can be observed to adjust the rate of API calls according to performance degradation over a period of time.

</Update>

<Update label="21st July 2022" description="">

- Fixed an issue where events could be created without a timezone. If no timezone is specified, it now falls back to the timezone stored against the current company.
- Added `itemName` and `itemId` to the `/order` endpoint's order item response data.
- Added `refundedAmount`, `isValid`, `requiresApproval`, `approvedAt`, `rejectedAt`, and `cancelledAt` to any order item response data inside orders.
- Added cursor pagination to the `/orders` endpoint, allowing for a streamed pagination approach, based on the `lastUpdated` timestamp provided.
- Added a `contains` parameter to the `/orders` endpoint, allowing you to filter orders that contain only the specified item types.
- Added `customerSsoId` parameter to POST `/basket-items` and POST `/orders/complete`
- Improved feedback & error handling on POST `/order-item/approve`
- Allow disabling both the order confirmation AND the guest list approved confirmation in POST `/basket-items`

</Update>
